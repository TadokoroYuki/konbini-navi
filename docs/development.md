# 開発ガイド

## 実機テスト

### Expo開発サーバーの起動

#### LANモード（デフォルト）

```bash
cd apps/mobile
npx expo start
```

Expo GoアプリでQRコードをスキャンして接続します。

> **注意**: PCとスマートフォンが同じWi-Fiネットワークに接続されている必要があります。

#### Tunnelモード（推奨）

LAN接続でタイムアウトやネットワークエラーが発生する場合は、tunnelモードを使用してください。

```bash
npx expo start --tunnel
```

tunnelモードでは `@expo/ngrok` を使って外部トンネル経由で接続するため、ネットワーク環境に依存しません。

初回実行時に `@expo/ngrok` のインストールを求められた場合は `y` で承認してください。

### LAN接続がタイムアウトする場合の対処法

1. **tunnelモードに切り替える**（最も簡単）
   ```bash
   npx expo start --tunnel
   ```

2. **ファイアウォールの確認**
   - macOS: システム設定 → ネットワーク → ファイアウォール で、Node.jsの接続を許可
   - Windows: Windows Defender ファイアウォールで Node.js を許可

3. **同一ネットワークの確認**
   - PCとスマートフォンが同じWi-Fiに接続されているか確認
   - 企業ネットワークやゲストWi-Fiではデバイス間通信がブロックされる場合があります

4. **ポートの確認**
   - Expoはデフォルトでポート `8081` を使用します
   - 他のアプリケーションが同じポートを使用していないか確認してください

5. **キャッシュのクリア**
   ```bash
   npx expo start --clear
   ```
